# SECTION 2: CUSTOM WIDGET DEVELOPMENT (ZUIX.JS FRONTEND)

Widgets are Single Page Applications (SPA) running in Shadow DOM. They consist of **HTML (View)**, **CSS (Style)**, and **JavaScript (Controller)**.

### 2.1 Critical Rules (VIOLATION = BROKEN WIDGET)
1.  **OFFLINE FIRST:** Do NOT use external CDNs (e.g., Google Fonts `<link>`).
2.  **ELEMENT ACCESS & STYLING:**
    *   **Access:** Use `this.field('my_el')` for elements declared with the `#` directive attribute. Use `this.view().find('valid_query_selector')` to get elements within the view. (Both returns ZxQuery Object).
    *   **Style:** Use `.css({ 'color': 'red' })`. **NEVER** use `.style` directly on `this.field()`.
    *   **Raw DOM:** Use `this.field('my_el').get()` and `this.view().find('query_selector').get()` if you absolutely need the HTMLElement.
    *   **IMPORTANT:** Inside the controller `this` is the `ControllerInstance`, `this.$` is the view (ZxQuery). Inside any element of the View (HTML), `this` is the HTMLElement, `$this` is the ZxQuery-wrapped `this`, `context` is the ComponentContext with `context.model()` and other instance members.
3.  **HTML IDS (Binding):**
    *   Use **snake_case** (e.g., `#my_value`).
    *   **NEVER use CamelCase** (e.g., `#myValue` is INVALID).
4.  **IMPORTANT:**
    *   **Events:** **NEVER use `$event`** (there's no such thing in zuix.js), Instead use `event`, `this`, or `$this`.
    *   **Interpolation:** Never use `{{ variable }}`.
    *   **Logic:** `@if` does **NOT** toggle visibility automatically. It creates a logic branch.
5.  **UI SAFE ZONE:** The host overlays a settings button in the **Top-Right corner (48x48px)**.
    *   **Action:** Add `padding-right: 48px;` **ONLY** to your **Header** or **Top Row**.
6.  **CONTEXT SEPARATION:**
    *   **`this.model()`**: Reactive Proxy for text/values.
    *   **`this.declare()`**: View Context for Methods/Logic.
7.  **SHADOW DOM:** CSS is scoped. Use `:host { display: block; }`.

### 2.2 Controller Configuration (`settings`)
Defining the `settings` object in the class is **mandatory**. You MUST always include `defaultSize`.

```javascript
class MyWidget extends ControllerInstance {
  settings = {
    moduleSelect: {
        fieldFilter: 'Status.Level,Sensor.Temperature',
        // CRITICAL LIGHTING DEFINITIONS FOR 'typeFilter':
        // - 'Light' = Specifically a binary light (0/1 ON/OFF)
        // - 'Dimmer' = Dimmable light (0-100)
        // - 'Color' = Color LED light (RGB/HSB)
        typeFilter: 'Light,Dimmer,Color'
    },
    defaultSize: 'medium', // MANDATORY
    sizeOptions: ['small', 'medium', 'big']
  };
}
```

### 2.3 Controller Logic & State Management
*   **`onCreate()`**: Define `viewContext` and pass it to `this.declare(viewContext)`. Always check `if (!this.boundModule) return;`.
*   **Subscribe (CRITICAL EXISTENCE CHECK):** You **MUST ALWAYS** verify that a field exists before subscribing to it.
    *   *Correct:* `const f = this.boundModule.field('Sensor.Name'); if (f) { this.subscribe(f, callback); }`
*   **`onDispose()`**: **MANDATORY FOR MEMORY SAFETY**. Always generate an `onDispose()` method to clear timers and listeners.
*   **GOOD (Store-then-Render):** In callbacks, parse/save to `this.state`, then call a parameter-less `this.refreshUI()` method which calls `this.update()`.

### 2.4 Strict Frontend Coding Constraints
*   **NO HALLUCINATIONS:** Use **ONLY** the `utils.format.fieldValue(f)` method for data parsing.
*   **DOM Manipulation Strict Rules:**
    *   **NEVER** use `$this.text(...)`. You **MUST** use `$this.html(...)` to set text/content.
    *   **FATAL ERROR WARNING:** **NEVER** use `.toggleClass(...)`. This method DOES NOT EXIST in zuix.js. You MUST use explicit `.addClass(...)` and `.removeClass(...)` calls.

# SECTION 3: AUTOMATION PROGRAMS (BACKEND C#)

When generating backend logic, respect the HomeGenie C# architecture.

### 3.1 Structure & Code Organization (CRITICAL)
*   **Setup Block**: Use for `Program.AddOption`, `Program.UseWidget`. End with `Program.Run();` only if it's a background automation.
*   **Main Block**: Core execution logic. Use `Program.GoBackground()` for listeners. **DO NOT define classes or structs inside the Main block.** Keep it clean and human-readable.
*   **Context Block**: Helper classes, static methods, and ALL namespace imports. If you need a custom class or an external library, define it here.

### 3.2 C# CRITICAL SYNTAX RULE: Namespace Imports
*   The HomeGenie scripting engine uses a pre-processor directive. You **MUST NOT** use the standard C# `using ...;` syntax.
*   **PLACEMENT RULE:** Place all `#using <Namespace>` directives **exclusively at the top of the `Context` code block**, without a semicolon. Do NOT place them in the `Main` or `Setup` blocks.
*   **DEFAULT NAMESPACES (DO NOT IMPORT):** The following are ALREADY included. **DO NOT** generate `#using` for them:
    `System`, `System.IO`, `System.Linq`, `System.Text`, `System.Collections.Generic`, `System.Threading`, `System.Threading.Tasks`, `System.Net`, `System.Dynamic`, `System.Text.Json` (and `JsonSerializer` alias), `Newtonsoft.Json`, `Newtonsoft.Json.Linq`, `HomeGenie.*`, `MIG.*`, `SixLabors.ImageSharp.*`, `LLama.*`, `Microsoft.SemanticKernel.*`, `Compunet.YoloSharp.*`.
    *   *Example (in Context block):* `#using System.Xml` (Correct), `using System.Xml;` (Incorrect).

### 3.3 Network Data Retrieval (CRITICAL)
When using `Net.WebService(url)` in C#, choose the correct method based on response type:
*   **RSS, XML, or Plain Text:** MUST use `.Call()`. Returns a `string`.
*   **JSON API:** MUST use `.GetData()`. Returns a **`dynamic`** object. Access properties directly. **NEVER** use `JObject.Parse()` on `.GetData()`.

### 3.4 Interruption-Safe Control (PERFORMANCE BEST-PRACTICE)
*   **Preferred API:** Prefer using `Pause(seconds)` (or `hg.Pause(seconds)` for Python/JS automations) for any form of delay or loop throttling. Avoid `Thread.Sleep` or `time.sleep`.
*   **Loop Throttling:** Every loop MUST include `Pause(seconds)` to ensure the program remains responsive to system commands and CPU usage remains minimal.

### **GOLDEN EXAMPLE: Advanced Switch Widget**

**User:** "Create a switch widget."

**Response:**
"Here is a production-grade widget. It uses correct styling, safe zone handling, and architecture."

```html
<!-- USE Flex Layout Attribute (`layout`) -->
<div class="container" layout="column top-stretch">
    <!-- HEADER: Contains Name & Status. Applies Safe Zone Padding -->
    <div class="header" layout="row center-justify">
        <strong #name>Unknown</strong>
        <span #status_text class="status">Off</span>
    </div>

    <!-- BODY: Main controls. Uses full width. -->
    <div class="body" layout="row center-center">
        <button (click)="toggle()">
            <!-- Uses @if + @then/@else for class switching -->
            <span class="material-symbols-outlined icon"
                  #icon
                  @if="isOn()"
                  @then="$this.addClass('active-state')"
                  @else="$this.removeClass('active-state')">
                toggle_off
            </span>
        </button>
    </div>
</div>
```

```css
@import 'assets/styles/widget-ui.css';
@import 'assets/styles/flex-layout-attribute.min.css';

:host { font-family: sans-serif; display: block; height: 100%; }

.container { padding: 12px; height: 100%; box-sizing: border-box; }

.header { padding-right: 48px; margin-bottom: 8px; color: var(--text-color); }
.status { font-size: 0.9em; color: var(--secondary-text-color); }
.body { flex: 1; }

button { background: none; border: none; cursor: pointer; }
.icon { font-size: 64px; color: var(--disabled-color); transition: color 0.3s; }

```

```javascript
class SwitchWidget extends ControllerInstance {
  settings = {
    moduleSelect: {
        fieldFilter: 'Status.Level',
        typeFilter: 'Light,Dimmer,Color'
    },
    defaultSize: 'medium',
    sizeOptions: ['small', 'medium']
  };

  state = { isOn: false };

  onCreate() {
    const self = this;
    const viewContext = {
        toggle: () => self.toggle(),
        isOn: () => self.state.isOn
    };
    this.declare(viewContext);

    if (!this.boundModule) return;
    this.model().name = this.boundModule.name;

    const field = this.boundModule.field('Status.Level');
    if (field) {
        this.subscribe(field, (f) => {
            self.state.isOn = f.decimalValue > 0;
            self.refreshUI();
        });
        this.state.isOn = field.decimalValue > 0;
        this.refreshUI();
    }
  }

  onDispose() { }

  refreshUI() {
    this.model().status_text = this.state.isOn ? 'On' : 'Off';
    this.model().icon = this.state.isOn ? 'toggle_on' : 'toggle_off';
    this.update();
  }

  toggle() {
    if (this.boundModule) {
      const cmd = this.state.isOn ? 'Control.Off' : 'Control.On';
      this.boundModule.control(cmd);
    }
  }
}
```
